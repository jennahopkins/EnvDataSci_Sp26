Our goals as ecological and earth system data scientists are to extract meaning from big environmental data and to provide valuable insight to stakeholders. This can only be done through careful work with data at every stage in the data life cycle.

Definitions

Data validation - process of ensuring that data are accurate (i.e., have correct values) and precise (i.e., have consistent values)

Quality assurance (QA) - systematic prevention of defects in a product or service so that they meet specified requirements

  • proactive approach
  • requires audit of the process or input

Quality control (QC) - identification and rectification of defects in a product or service so that they meet specified requirements

  • reactive approach
  • requires audit of the product or output

Quality Assurance Considerations

The primary goal of QA is to scrub the errors from the input to the analytical pipeline.

  1. Consider the format of the raw data:
  • relational (i.e., tabular) vs.non-relational (e.g., document, graph) data
  • row count and meaning
  • column count and meaning
  • data types (e.g., categorical or numerical)
  • file type

Quality Control Considerations

The primary goal of QC is to scrub the errors from the output from the analytical pipeline.

  1. Consider future data users:
  • Do the metadata explicitly explain basic information about the data that can make tracking and working with the data easier?
  • Will other scientists be able to interpret my summary figures and tables?
  • Will other scientists or future me be able to read and understand my code?
  1. Consider the stakeholders:
  • Will non-scientist subject matter experts be able to interpret my summary figures and tables?
  • Do my products contain proprietary information that must be redacted?

NOAA Data Example

Well work with a simple dataset from the National Oceanic and Atmospheric Administration (NOAA). These data are the bimonthly values for the Multivariate El Nio Southern Oscillation Index (https://psl.noaa.gov/enso/mei/) or MEI for short. Our end goal is to create a time series showing the El Nio and La Nia events over the past 47 years.

First, lets load the input data. There are four versions of the data:

  • raw data updated on March 10, 2025
  • cleaned data updated on March 10, 2025
  • raw data updated on January 8, 2026
  • cleaned data updated on January 8, 2026

Of course, these data actually represent a time series, which is best plotted using a line graph. We need to wrangle our dataset to plot a time series, which well do using functions from the R packages tidyr, dplyr, and ggplot2.

The dataset is currently in wide format, meaning there are redundant variable types across the columns. We need to convert it to long format, so that the same variables are stacked in a single column. Well use the function pivot_longer to achieve this. The important arguments are:

  • cols for the columns to pivot into longer format
  • names_to to specify the new column to create from the information stored in cols
  • values_to to specify the new column to create from the data stored in the cell values.
MEI_long <- MEI %>%
  pivot_longer(cols = DJ:ND, names_to = "Month", values_to = "MEI")

We have two new products: a dataframe called MEI_long and a time series plot called MEIseries. Now lets follow the principles of quality control to improve these data products!

First, lets make our dataset more meaningful by categorizing the bimonthly MEI as El Nio (MEI \(\ge 0.5\)), neutral (\(-0.5 <\) MEI \(< 0.5\)), or La Nia (MEI \(\le -0.5\)). Well make a new column to store the ENSO phase using mutate and nested if_else statements. Lets make sure our logic was applied correctly using the table function.

MEI_long <- MEI_long %>% 
  mutate(ENSOphase = if_else(MEI < -0.49, "La Nina",
                             if_else(MEI > 0.49, "El Nino",
                                     "Neutral")))

Because journals do not typically want the horizontal or vertical grid lines on plots, theme_classic is the best choice here. But there are important thresholds in these data, so well add three horizontal lines using geom_hline.

MEIseries + theme_classic() +
  geom_hline(yintercept = 0, linetype = "solid") +
  geom_hline(yintercept = 0.5, linetype = "dashed") +
  geom_hline(yintercept = -0.5, linetype = "dashed")

Now we can see when the time series was in the three ENSO phases, but lets add some colored columns to make it more obvious! Well also remove the missing data from MEI_long using filter.

ggplot(data = filter(MEI_long, !is.na(MEI)), aes(x = Date, y = MEI)) +
  geom_line() + theme_classic() +
  geom_hline(yintercept = 0, linetype = "solid") +
  geom_hline(yintercept = 0.5, linetype = "dashed") +
  geom_hline(yintercept = -0.5, linetype = "dashed") + 
  geom_col(aes(x = Date, y = MEI, fill = ENSOphase))

The default colors dont match our intuition of El Nio being warm and La Nia being cool, so lets change these using scale_fill_manual. Well also use the character code for (\u00F1). In addition, lets make the black line bigger and draw it on top of the columns.

ggplot(data = filter(MEI_long, !is.na(MEI)), aes(x = Date, y = MEI)) +
  theme_classic() +
  geom_hline(yintercept = 0, linetype = "solid") +
  geom_hline(yintercept = 0.5, linetype = "dashed") +
  geom_hline(yintercept = -0.5, linetype = "dashed") + 
  geom_col(aes(x = Date, y = MEI, fill = ENSOphase)) +
  geom_line(linewidth = 0.7) +
  scale_fill_manual(name = "ENSO Phase",
                    values = c("dodgerblue","gray36","firebrick"),
                    labels = c("La Ni\u00F1a","Neutral","El Ni\u00F1o"))

Were almost done, but lets improve the x-axis and y-axis with better titles and breaks. We also want to put the legend above the figure and make all the font black.

ggplot(data = filter(MEI_long, !is.na(MEI)), aes(x = Date, y = MEI)) +
  theme_classic() +
  geom_hline(yintercept = 0, linetype = "solid") +
  geom_hline(yintercept = 0.5, linetype = "dashed") +
  geom_hline(yintercept = -0.5, linetype = "dashed") +

                    labels = c("La Ni\u00F1a","Neutral","El Ni\u00F1o")) +
  theme(legend.position = "top", 
        axis.text = element_text(color = "black"))
